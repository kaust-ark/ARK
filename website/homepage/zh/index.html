关于 ARK

从想法到论文，全自动

ARK（Automatic Research Kit）是一个从想法到论文的科研自动化服务。给它一个研究想法和目标会议——ARK 负责文献调研、实验设计、代码执行、图表生成、LaTeX 撰写和迭代修改。六个专业 AI Agent 通过结构化流程协作，您可以随时通过 Web Dashboard 或 Telegram 掌控每一个关键节点。

自部署

一行命令安装

支持 Linux 与 macOS。脚本会构建 conda 环境，装好 ARK + Claude Code + Gemini CLI，把仪表板装成 systemd --user 服务，并打印一个一次性的 magic-link 链接让你本地登录。不需要 SMTP，也不需要 Google OAuth。

curl -fsSL https://idea2paper.org/install.sh | bash
--no-webapp 跳过本地仪表板 --no-research 跳过可选研究依赖 --prefix DIR 安装到自定义目录 --dry-run 仅预览不动手

脚本会提示输入 Gemini API key、Claude OAuth token、登录邮箱。装完点提示里的 magic link → 直接进 http://localhost:9527 仪表板。完整文档见 doc.html · 脚本见 install.sh。

其他工具
  • 自动化漂移
    无人监督运行，中途无法纠偏。
  • 排版混乱
    布局错乱、LaTeX 报错，需大量人工修复。
  • 引用幻觉
    LLM 编造看似合理但不存在的参考文献。
  • 图片质量低
    默认样式、尺寸失控，无视栏宽和页面约束。
  • 配置繁琐
    需手动配置环境、调整依赖，上手门槛高。
ARK
  • 人机协同
    关键决策点暂停，随时通过 Telegram 或网页介入。
  • 出版级排版
    硬编码 LaTeX + 会议模板（NeurIPS、ACL、IEEE……）。
  • 引用验证
    每条引用经 DBLP API 验证，杜绝虚假文献。
  • 精准控图
    Paper Banana + 会议画布尺寸、栏宽、字号精确匹配。
  • 一键启动
    网页端一键运行，零配置开箱即用。

工作原理

ARK Pipeline

研究、开发、审稿——持续迭代直至达到目标分数。

1

研究

Deep Research 搜文献、理背景。

2

开发

定方案、跑实验、分析结果、写初稿。

3

审稿

迭代优化，直到审稿分数达到阈值。

研究阶段

⚙️ 环境搭建 → 📖 分析提案 → 🔬 深度调研 → 🎯 特化定制 → 📚 引用引导

开发阶段

迭代式
📋 规划实验 → 🧪 执行实验 → 📊 分析结果 → ✅ 完整性评估 → 📈 生成图表 → ✍️ 撰写初稿 → 📦 交付

审稿阶段

直到分数 ≥ 阈值
🔍 审稿 → 📋 规划 → ⚡ 执行

认识团队

ARK Agents

六个专家 Agent，通过共享状态和结构化交接协作。

Researcher

分析提案，生成深度调研查询，引导引用。

Reviewer

逐页评估论文，在多个维度上打分。

Planner

分类问题，创建修复任务，规划实验。

Writer

修改 LaTeX 论文和 Python 绘图脚本。

Experimenter

运行真实实验，收集并分析结果。

Coder

编写生产代码、测试和维护代码质量。
